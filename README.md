# How Dev — 开发者技术全景指南

## 这是什么

面对一个空白编辑器和新语言/技术栈时，第一步不是学语法——而是理解这片领域的"地图"：工具链关系、生态位、编码惯例、从源码到产物的完整路径。

本仓库提供一系列**技术全景文档**，每份文档覆盖一个语言或领域的完整开发生态，让你在动手写代码之前先建立心智模型。

## 文档结构

```
how-dev/
├── topics/                    # 跨语言通用主题
│   ├── compilation.md         # 编译通识：预处理→编译→汇编→链接
│   └── package-management.md  # 包管理通识：依赖解析、锁文件、注册中心
├── languages/                 # 按语言
│   ├── c.md                   # C 语言全景指南
│   ├── python.md              # Python 全景指南
│   ├── go.md                  # Go 全景指南
│   ├── rust.md                # Rust 全景指南
│   └── javascript.md          # JavaScript / TypeScript 全景指南
├── domains/                   # 按领域（跨语言）
│   ├── frontend.md            # 前端开发全景指南
│   └── backend.md             # 后端开发全景指南
└── requirements.md            # 项目需求文档
```

## 每份文档覆盖什么

每份语言全景文档统一覆盖 **10 个维度**：

1. **语言画像** — 编译/解释？类型系统？内存模型？编程范式？
2. **源码到运行** — 从代码到可执行产物的完整流程
3. **工具链地图** — 工具之间的替代/互补/上下游关系（含 Mermaid 图）
4. **依赖管理** — 包从哪里来？怎么管版本？锁文件干什么？
5. **项目结构** — 社区约定俗成的目录和文件组织方式
6. **编码惯用法** — 错误处理、资源管理、并发模型的典型模式
7. **测试版图** — 单元测试、mock、benchmark、覆盖率的主流工具
8. **部署分发** — 产物形态、交叉编译、发布渠道
9. **代表性项目** — 值得读源码的知名开源项目
10. **实用入门路径** — 最小环境搭建到第一个可运行项目

## 适合谁

- 已有编程基础，需要快速切入新语言的开发者
- 使用 Neovim/Helix/Emacs 等轻量编辑器，需自行搭建开发环境
- 希望理解技术生态全貌而非停留在语法层面的学习者

## 不适合谁

- 零基础学编程（请先学会一门语言）
- 需要详细的 API 参考或工具手册（请查阅官方文档）
- 需要面试题、算法题解

## 使用方式

从头开始：选择你要切入的语言或领域，先读 `topics/compilation.md`（如果是编译型语言）和 `topics/package-management.md`，再读对应的语言文档。每篇文档约 10-15 分钟可读完。

作为参考：当你遇到"该选哪个包管理器"、"代码应该放在哪个目录"、"怎么测试"这类问题时，回到对应文档查阅相关维度。
